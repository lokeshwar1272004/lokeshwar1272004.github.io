// PROJECT 01
Hybrid Knowledge Graph + RAG QA System
Ask any question about your private documents — zero hallucination, full offline support.
⌥ GitHub ▶ Demo Video
// Problem & Solution

LLMs hallucinate on private documents. Enterprise users can't afford wrong answers from internal reports or research papers. Existing RAG systems miss relationship-based questions that require graph traversal.

Built a hybrid pipeline that routes every question to the right retrieval method — GRAPH for relationships, VECTOR for semantics, or HYBRID for both — automatically, using an LLM-based query classifier.

// Key Features
  • PDF / TXT / DOCX ingestion in one unified pipeline
  • spaCy NER + LLM-based entity & relation extraction
  • Neo4j Knowledge Graph + ChromaDB Vector Store
  • Auto query classifier → GRAPH / VECTOR / HYBRID
  • Groq API with automatic Ollama local fallback
  • Interactive PyVis Knowledge Graph visualisation
  • Confidence scoring on every answer
// Impact & Results

// Architecture
  • Document → spaCy NER → Neo4j graph construction
  • Document → sentence-transformers → ChromaDB index
  • Query → LLM classifier → retrieval path selection
  • Retrieved context → Groq / Ollama → grounded answer
  • diskcache layer + tenacity retries for resilience

// PROJECT 02
Customer Churn Prediction
End-to-end ML pipeline predicting telecom churn with SMOTE-balanced XGBoost and feature insights.
⌥ GitHub
// Problem & Solution

Telecom companies lose significant revenue to churn, but identifying at-risk customers manually is slow, error-prone, and unscalable across millions of records.

Trained and compared Logistic Regression vs XGBoost classifiers on real telecom data, applying SMOTE to fix class imbalance. Selected the best model by F1-score and built feature importance visualisations to support business decisions.

// Pipeline Steps
  • Missing value handling & categorical encoding
  • SMOTE oversampling for class imbalance
  • Logistic Regression vs XGBoost model comparison
  • Accuracy, Precision, Recall, F1-score evaluation
  • Feature importance & churn pattern visualisations
// Key Results
  • SMOTE significantly improved minority class recall
  • XGBoost outperformed baseline Logistic Regression on F1
  • Feature importance charts actionable for retention teams
  • Confusion matrix analysis surfaced false-negative cost
  • Full reproducible notebook — clean, documented code
// Learnings
  • Class imbalance strategy matters more than model choice
  • Precision vs recall trade-off is a business decision, not a metric
  • Feature engineering surfaces patterns hidden from raw columns
